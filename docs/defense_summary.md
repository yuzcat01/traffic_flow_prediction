# 答辩结论与讲解提纲

## 1. 一分钟结论版

如果答辩时需要先快速给出结论，可以直接这样说：

> 本项目面向交通流量多步预测任务，设计并实现了一个基于图神经网络和时序建模的本地可运行预测系统。系统支持数据预处理、模型训练、模型管理、推理分析、结果可视化和报告导出。实验部分对 `GCN-GRU`、`ChebNet-GRU` 和 `GAT-GRU` 三类模型进行了 `h3`、`h6` 和 `h12` 三组多步预测对照。结果表明，`GAT-GRU` 在短步长下精度略优，但计算开销较大；`ChebNet-GRU` 在精度和效率之间取得了最佳平衡，并在 `h12` 长步长任务上取得了更优的总体 `RMSE`，因此最终将 `ChebNet-GRU` 作为系统主推模型。

## 2. 三分钟项目介绍版

可以按下面顺序讲：

1. 项目目标
   构建一个面向城市交通流量预测的可视化系统，实现数据管理、图神经网络建模、多步预测和结果分析。
2. 技术路线
   采用图神经网络提取空间依赖，采用 `GRU` 建模时间依赖，并通过统一输出头实现未来多步预测。
3. 系统功能
   系统已经支持数据预览、缺失值处理、异常值裁剪、训练监控、模型管理、推理分析、结果对比和报告导出。
4. 实验结论
   对 `GCN`、`ChebNet`、`GAT` 三种空间模块进行 `h3/h6/h12` 对照实验后，综合精度和效率，选择 `ChebNet-GRU` 作为主模型。

## 3. 五分钟实验结论版

### 3.1 为什么要做三模型对比

可以这样说：

> 由于交通流量预测同时依赖道路网络结构和时间序列变化，不同图神经网络空间模块会直接影响预测效果和运行成本。因此我选择了结构简单的 `GCN`、频域卷积思想更强的 `ChebNet` 以及注意力机制更强的 `GAT` 进行对照实验，以便找到最适合本系统的模型方案。

### 3.2 为什么最后选择 `ChebNet-GRU`

可以这样说：

> 实验结果表明，`GAT-GRU` 在部分短步长任务中略有精度优势，但训练和推理时间显著更高。`ChebNet-GRU` 在 `h3` 和 `h6` 下与 `GAT-GRU` 的指标非常接近，在 `h12` 长步长任务上甚至取得了更优的总体 `RMSE`。同时它的训练和推理速度明显优于 `GAT-GRU`，因此更符合本项目在本地环境下运行、展示和复现实验的需求。

### 3.3 如何回答“为什么不是 GAT”

可以直接回答：

> 不是因为 `GAT` 不好，而是因为本项目不仅追求精度，还要考虑本地运行效率、实验复现成本和系统响应速度。当前结果显示，`GAT` 的精度提升非常有限，但时间成本明显更高，因此综合性价比不如 `ChebNet`。

### 3.4 如何回答“为什么不是 GCN”

可以直接回答：

> `GCN` 更适合做基线模型。它结构清晰、容易实现，但在 `h3`、`h6` 和 `h12` 三组实验中都明显落后于 `ChebNet` 和 `GAT`，因此更适合作为对照组，而不是最终主模型。

## 4. 可能被问到的问题与回答

### 4.1 为什么做多步预测

可答：

> 单步预测只能反映最近一个时刻的预测能力，而实际交通管理更关心未来一段时间内的变化趋势，因此需要做 `h3`、`h6`、`h12` 这样的多步预测，以评估模型在短期、中期和较长期预测任务上的稳定性。

### 4.2 为什么图展示不能只看 node0 和 horizon1

可答：

> 只看单个节点和单个步长会忽略大量时空信息，所以我后来把结果展示扩展成了多节点代表图、全步长趋势图和节点-步长误差热力图，这样展示口径更完整，也更符合真实评估需求。

### 4.3 系统做了哪些工程优化

可答：

> 主要做了三类优化。第一是训练和推理链路优化，比如批量化空间编码、减少重复图传输、增加吞吐统计和数值保护。第二是 GUI 优化，比如多步指标展示、筛选排序、结果对比和报告导出。第三是文档和复现能力优化，比如补充需求说明、技术设计、字段说明、运行说明和实验结果文档。

### 4.4 项目还有哪些可以继续完善

可答：

> 后续还可以继续补多随机种子重复实验、完善统计显著性分析、优化 `GAT` 前向效率，并考虑引入更强的时间建模模块或动态图建模方式。
