# 交通流量预测系统需求说明

## 1. 项目目标

本项目面向“基于图神经网络的交通流量预测系统的设计与实现”毕业设计课题，目标是构建一个可复现、可扩展、可视化的交通流量预测实验平台。系统既要支持模型研究，也要支持演示与应用验证。

## 2. 功能需求

### 2.1 数据管理

- 支持导入图结构文件与交通流量文件。
- 支持生成数据配置文件并保存到 `configs/data/`。
- 支持数据集摘要展示，包括节点数、时间步、训练/测试划分、样本量等。
- 支持图结构统计展示，包括邻接矩阵规模、边数、稀疏度等。
- 支持节点时间序列预览。
- 支持时空热力图预览。
- 支持预览摘要与节点序列导出。
- 支持基础预处理配置，包括缺失值处理和异常值裁剪。

### 2.2 模型训练与评测

- 支持 `GCN`、`ChebNet`、`GAT` 三类空间建模模块。
- 支持统一时间模块框架，当前已支持 `GRU` 和 `none`。
- 支持多种建图方式，包括 `connect`、`distance`、`correlation`、`distance_correlation`。
- 支持单步与多步预测。
- 支持训练、验证、测试流程。
- 支持学习率调度、梯度裁剪、早停策略与多种损失函数。
- 支持实验配置保存、模型权重保存、预测曲线保存和结果汇总。

### 2.3 模型管理与推理

- 支持实验记录浏览、排序、筛选和导出。
- 支持加载最优模型或指定模型。
- 支持在线推理与样本级结果分析。
- 支持批量评估并导出推理结果。

### 2.4 结果分析与报表

- 支持当前模型指标展示。
- 支持多实验分组对比。
- 支持多模型对比与步长对比。
- 支持报告包导出，包括图表、CSV 和 Markdown 报告。

## 3. 非功能需求

- 可读性：模块边界清晰，配置命名统一，页面职责明确。
- 可维护性：训练、预测、数据处理、GUI 解耦，便于独立修改。
- 可扩展性：允许继续增加新的图模型、时间模块、预处理策略和可视化组件。
- 可复现性：关键实验参数、运行配置、结果指标和图像文件可保存。
- 响应速度：训练和推理链路避免不必要的数据复制和重复计算。
- 稳定性：数据配置、预测步长、样本数量等异常情况要给出明确报错。
- 本地运行友好：支持在 conda 环境中完成训练、GUI 启动和实验复现。
