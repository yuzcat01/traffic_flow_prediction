# 应用场景模块说明

## 1. 模块定位

应用场景模块用于把交通流量预测结果转化为可解释的交通辅助决策。系统在加载训练好的模型后，可以从测试集滑动窗口中回放一段历史流量，模拟实时路网数据输入，再预测未来若干时间步的节点流量。

在此基础上，系统结合路网拓扑边距离计算路线代价，为用户提供三个应用页面：

- 路线规划：多候选路线推荐、可达性查询、路网拓扑预览和路线高亮。
- 拥堵预警：全网未来风险 Top-K、严重风险节点统计和预警建议。
- 事件模拟：对指定节点施加事件影响，比较事件前后的推荐路线变化。

## 2. 核心流程

1. 用户在“模型管理”页面加载已训练模型。
2. 系统复用 `TrafficPredictor` 完成未来节点流量预测。
3. `RouteRecommendationService` 读取图结构文件中的 `from / to / cost` 边信息。
4. 系统用历史流量分位数作为节点参考流量，计算拥堵指数。
5. 按用户选择的策略计算路径边权，并使用 Dijkstra 算法生成推荐路线。
6. “路线规划”页面展示多条候选路线、节点风险、全网高风险节点和导出报告。
7. 系统根据图结构连接关系绘制路网拓扑预览，并在用户切换候选路线时高亮当前路线。
8. 用户可以单独查询起点和终点是否可达，系统会返回是否连通、最少连接跳数和一条连接路径。
9. “拥堵预警”页面按预测步长统计全网高风险节点，形成预警摘要。
10. “事件模拟”页面修改指定节点的预测流量，重新计算路线并展示事件影响。

## 3. 拥堵指数

节点拥堵指数定义为：

```text
congestion_score(node) = predicted_flow(node) / baseline_flow(node)
```

其中 `baseline_flow` 默认采用该节点历史流量的 85% 分位数。这样可以避免直接假设道路容量，也更适合 PeMS04 这类传感器节点数据集。

页面和图表中的“参考流量”就是这里的 `baseline_flow`。它来自当前数据集的历史流量统计，用于把预测流量转成拥堵指数；它不是道路设计容量，也不是外部实时交通系统提供的限流值。

拥堵等级划分：

| 拥堵指数 | 等级 |
|---|---|
| < 0.55 | 畅通 |
| 0.55 - 0.75 | 轻度 |
| 0.75 - 1.00 | 中度 |
| >= 1.00 | 严重 |

## 4. 路线推荐策略

当前支持三类策略：

| 策略 | 说明 |
|---|---|
| 距离最短 | 仅考虑图结构中的边距离 |
| 优先避堵 | 主要考虑未来预测拥堵风险 |
| 综合最优 | 同时考虑边距离和预测拥堵风险 |

综合最优策略的边权形式为：

```text
edge_weight(u, v) = distance(u, v) * (1 + alpha * avg_congestion(u, v))
```

其中：

```text
avg_congestion(u, v) = (score(u) + score(v)) / 2
```

`alpha` 是拥堵权重，值越大，路线越倾向于绕开预测拥堵节点。

## 5. 路网预览与交互

路线规划页面会基于图结构文件中的连接关系构建整体路网预览。由于默认 PeMS04 数据不包含真实经纬度，页面使用连接关系生成拓扑布局，用于展示节点之间的相对连接结构，而不是还原真实地图坐标。

交互能力包括：

- 同时生成多条候选路线，用户可以在候选路线表中切换方案。
- 路网图会用不同颜色展示候选路线，并加粗高亮当前选中的路线。
- 对当前路线和最短距离路线做距离、平均拥堵、最高拥堵、高风险节点数对比。
- 根据对比结果生成推荐原因，说明是否绕开高风险节点、是否降低平均拥堵以及距离增加多少。
- 当前起点和终点会在图中以特殊标记显示。
- 点击路网节点可以快速改变起点，右键点击节点可以改变终点，页面会重新计算候选路线。

页面还提供“选择最远预测步长”按钮。对于 `predict_steps=12` 的模型，最远步长就是 `h12`；对于 `predict_steps=3` 的模型，最远步长就是 `h3`。这样用户不需要手动换算内部索引。

## 6. 拥堵预警页面

拥堵预警页面面向交通运行监控场景，核心目标是把未来某个预测步长下的节点流量转换成可快速阅读的风险列表。

页面会展示：

- 预测样本和预测步长对应的未来时间；
- 全网最高拥堵指数、Top-K 平均拥堵指数、严重风险节点数量；
- 高风险节点表格，包含预测流量、参考流量、拥堵指数和拥堵等级；
- 拥堵指数柱状图，用于快速识别风险集中在哪些节点；
- 中文预警摘要，给出是否需要重点关注、建议复核哪些节点。

这个页面不计算路线，而是服务于“全网态势感知”：调度人员可以先看哪些节点会出现风险，再进入路线规划或事件模拟页面做具体处置。

## 7. 事件模拟页面

事件模拟页面用于演示事故、施工、临时管制、突发需求上升或流量下降对路线推荐的影响。它不会改写原始数据，而是在当前预测结果上临时调整指定节点的预测流量。

当前实现方式：

```text
event_prediction(node) = original_prediction(node) * traffic_change_multiplier
```

其中 `node` 来自页面输入的事件节点列表，`traffic_change_multiplier` 是流量改变倍数。大于 `1` 表示流量放大，小于 `1` 表示流量缩小，等于 `1` 表示保持原预测流量。例如事件节点为 `8,9,14`、改变倍数为 `1.6`，表示这些节点在当前预测步长下的流量压力被临时放大 60%；改变倍数为 `0.6` 则表示缩小为原来的 60%。

页面会分别使用原始预测和事件预测计算路线，并对比：

- 原始推荐路线与事件后推荐路线；
- 距离、平均拥堵、最高拥堵和高风险节点数；图表会按距离、拥堵指数、风险节点数拆成独立纵轴，避免不同量纲互相压缩；
- 事件影响摘要和事件后路线推荐原因。

这个页面适合在答辩或演示中说明：预测模型不仅能给出未来流量，还能作为交通管理的“假设分析”工具，帮助判断某些节点异常后路线是否需要调整。

## 8. 路网图生成方法

路网预览图不是静态图片，而是由图结构文件动态生成：

1. 读取 `graph_path` 指向的 CSV 文件，要求至少包含 `from`、`to` 和 `cost` 字段。
2. 将 `from/to` 解析为节点连接关系，将 `cost` 作为边距离。
3. 因为默认数据没有真实经纬度，系统先把节点放在圆形初始位置上。
4. 再使用力导向布局思想迭代调整节点位置：
   - 所有节点之间有排斥力，避免节点完全重叠；
   - 有边连接的节点之间有吸引力，让相连节点更靠近；
   - 迭代结束后把坐标归一化到绘图区。
5. 画图时先画全部灰色路网边，再按拥堵指数给节点着色，最后叠加候选路线和当前选中路线。
6. 图例说明节点颜色、候选路线、当前路线、起点和终点含义。

因此，路网图表达的是“连接关系拓扑”，不是地图坐标。它可以用于观察网络结构、候选路线经过了哪些节点、哪些节点风险更高。

## 9. 路线计算算法

路线计算分两步：

第一步是把预测流量转成节点拥堵指数：

```text
congestion_score(node) = predicted_flow(node) / baseline_flow(node)
```

第二步是把节点拥堵指数转换为边权，然后搜索路径。

三种策略对应的边权如下：

```text
距离最短:
edge_weight(u, v) = distance(u, v)

优先避堵:
edge_weight(u, v) = 1 + alpha * avg_congestion(u, v)

综合最优:
edge_weight(u, v) = distance(u, v) * (1 + alpha * avg_congestion(u, v))
```

其中：

```text
avg_congestion(u, v) = (score(u) + score(v)) / 2
```

系统使用基于优先队列的 Dijkstra / Uniform Cost Search 思路搜索路径。单条最优路线就是最小总代价路径；多候选路线则是在不重复经过同一节点的前提下，按总代价从小到大继续展开路径，取前 N 条作为候选方案。

起终点可达性查询与路线推荐分开实现。可达性只判断图结构中是否存在连接路径，不考虑距离和拥堵权重，使用无权 BFS 搜索最少连接跳数；路线推荐则使用带权代价搜索。

## 10. 模拟数据集

为了验证应用场景模块不依赖 PeMS04，仓库提供了一个模拟数据集：

```text
data/raw/sim_demo/sim_demo.csv
data/raw/sim_demo/sim_demo.npz
configs/data/sim_demo.yaml
```

该数据集包含：

- 24 个模拟路网节点；
- 43 条连接边；
- 10 天 15 分钟粒度的节点流量；
- 早晚高峰、周末波动和局部临时拥堵扰动。

可以通过下面命令在模拟数据上训练模型：

```bash
python train.py --data_cfg configs/data/sim_demo.yaml --model_cfg configs/model/gcn_gru_h3.yaml
```

也可以直接运行命令行演示，验证路网预览、候选路线和推荐解释不依赖 PeMS04：

```bash
python scripts/demo_route_recommendation.py --data_cfg configs/data/sim_demo.yaml --source 0 --target 23 --candidate_count 3
```

只检查可达性：

```bash
python scripts/demo_route_recommendation.py --data_cfg configs/data/sim_demo.yaml --source 0 --target 23 --check_reachability
```

建议 GUI 演示流程：

1. 训练或加载 `SimDemo` 对应模型；
2. 打开“路线规划”页面；
3. 设置起点 `0`、终点 `23`；
4. 点击“选择最远预测步长”；
5. 策略选择“综合最优”，候选路线数选择 `3`；
6. 在候选路线表中切换方案，观察路网图高亮、对比表和推荐原因变化。
7. 打开“拥堵预警”页面，查看同一样本和预测步长下的全网风险节点。
8. 打开“事件模拟”页面，输入事件节点和流量改变倍数，对比路线是否发生变化。

## 11. 数据边界

PeMS04 默认数据包含传感器节点、边连接和边距离，但不包含真实经纬度、道路名称、限速、车道数等导航系统字段。因此本模块定位为：

```text
基于传感器节点拓扑的路线推荐原型
```

它适合展示交通流量预测模型如何服务于拥堵预警、路径辅助决策和交通管理演示；如果要扩展到生产级导航，还需要补充真实地图匹配、道路属性、实时事件、权限审计和外部数据接入能力。
